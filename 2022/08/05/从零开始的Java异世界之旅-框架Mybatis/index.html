从零开始的Java异世界之旅之SSM(一)

1. 简介–什么是mybatis

  1. mybatis是一个优秀的持久层框架（Dao层的框架）
  2. 支持定制SQL 存储过程以及高级映射
  3. 几乎避免了所有的JDBC代码和手动设置参数以及获取结果集
  4. mybatis可以通过xml或注解来配置和映射原生类型、接口和java的POJO

    2. 一个我自己的mybatis项目

  5. 创建一个基本的maven项目
    1. 在pom.xml内配置最基本的依赖
    2. 在resource内新建mybatis配置文件
    3. 创建实体类
    4. 创建实体类接口
    5. 配置文件连接Mapper
    6. 建立工具类MybatisUtils
    7. 建立UserMapper.xml

      3.CRUD

      对于<select><insert><delete>标签内的
  6. id:是应用该sql语句的方法
  7. parameterType:映射的是哪个Mapper（可以通过别名的方式来简化，不用每次都写路径）

    4.配置解析

    1. 核心配置文件(mybatis-config.xml内的配置)

  • configuration：字面意思了，就是配置标签
  • properties：可以用来连接外部properties文件，例如数据库的db.properties，
    这样可以省略掉在配置文件内编写数据库连接参数。
  • typeAlias:就是别名，根据提供的包名编写特定的别名，这样在mapper内就可以
    省略掉路径之类的。也可以通过<package>来自动扫描某个路径内的bean，写在parameterType内或者javaType内
  • typeHandler类处理
  • plugin插件
  • environments:可以设置多个环境并且设置id，通过更改default来修改sqlsession的环境。
  • mapper:将mapper注册为映射器，可以通过resource注册，也可以通过package把包下的所有mapper注
    册为映射器，还有class来注册，class注册的可以使用注解，更强大
  • setting: 主要记住两个子标签：cache缓存, logImpl日志

    1.1 问题

  1. mapper注册映射器，这个具体是干嘛的，什么是注册映射器？
    解答:把某个实体类的Mapper与这个对应的mapper.xml连接。因为mybatis通过映射
  2. typeHandler是干嘛的？（这玩意暂时用不上，不深纠）
  3. configuration的标签可以多写吗？它的作用是充当个填写标签的容器么
    解答:是的

2. 生命周期和作用域

5. 解决属性名(实体类内属性)和字段名(数据库内字段）不一致

  1. 用别名:即在mapper的select等标签内增加alias字段。
  2. 用resultMap:不使用resultType来确定返回类型，而使用resultMap来新建一个映射类型
    然后通过resultMap标签来为result设置值与字段匹配的内容。

    6.日志工厂

    这个部分学了有两种日志，一种是自带的STU啥的，一个log4j，log4j更强大一点，可以把日志存储在一个文件内
    使用方法是在mybatis-config.xml内编辑setting 在里面选择日志类型，但是使用log4j需要去添加log4j依赖

7. 分页

首先知道什么是分页
select * from user limit startIndex,pageSize
例如:select * from user limit 0,#[0,3];

  1. 我们mybatis的方式实现分页:
  • 首先是接口List<User> getUserByLimit(Map<String,Integer> map);
  • 然后是Mapper
  • 然后去把map里面的值put进去。
  1. 通过RowBounce来实现这个效果
  • 先是<select id="getUserByRowBounce resultMap="map"
  • 然后new RowBounce(1,2);
  • 然后在通过sqlsession使用selectList

8.使用注解开发

  1. 我们可以将mapper注入进config的mapper(通过class方式注入)
  2. 然后我们在对应的方法上我们使用@select(sql语句)的方式来直接编写sql语句
  3. 注意，这样的方式可以用在简单的sql语句上，但是哪些复杂的，需要resultMap映射的
    sql语句却无法实现，所以xml还是用的多点
  4. 这是通过什么样的机制完成的呢？
    • 反射机制
    • 底层其实是动态代理
    • 这里面涉及一个mybatis底层的流程
    • 在我们明面上的流程:resource获取全局配置文件->configuration获取全局配置信息
      ->sqlsessionFactory实例化->创建sqlsession->执行增删改查->sqlsession.close
    • 但是实际流程是: resource获取全局配置文件->实例化sqlSessionFactoryBuilder->解析配置文件流XMLconfigurationBuilder->
      configuration获取全局配置信息 ->sqlsessionFactory实例化->transactional事物管理->创建executor执行器->
      创建sqlsession->执行增删改查->sqlsession.close
