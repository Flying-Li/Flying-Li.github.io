二、类变量和方法

1.类变量和类方法

1.1 类变量？是什么？

有一群孩子在堆雪人，不时有新的小孩加入，那么该如何知道现在有多少小孩在玩？

显而易见的，我们的count是没办法在多个对象加入游戏的时候完成count自动++的，它是独立于对象存在的一个变量。如何解决这个问题，我们可不可以有一种变量，在每一个对象创建的时候都可以调用这个变量来修改变量？共享一个数据空间让每个类都可以访问这个空间就是类变量。

类变量出现了。

可以发现我们在Child内定义的类变量可以让这个类的对象都使用，并且都可以修改该变量。

1.2 类变量的内存分析

两种主流说法认为

  • 类变量存放在方法区内的静态域内（JDK7-)
  • 类变量存放在堆内类构件区块的末尾(JDK7+)

1.3 类变量定义和访问

类变量的定义

  • 访问修饰符 static 数据类型 变量名;(推荐)
  • static 访问修饰符 数据类型 变量名;

类变量的访问

  • 类名.类变量名(推荐)
  • 对象名.类变量名
1.4 类变量使用详解

  • 什么时候需要类变量？

    ​ 当我们需要让多个类共享访问同一个数据空间的时候我们可以定义类变量。

  • 类变量和实例变量的区别

    ​ 类变量是所有对象共享的，实例变量是单独对象特有的

  • 类变量在类加载时就已经初始化了，所以即使没有对象也可以被调用

  • 类变量随类加载而创建，随类销毁而消亡

1.5 类方法基本介绍

大致与类方法相同？🤨🤨🤨，也是所有对象都能共享到的方法，它有一个收学费的案例

类方法定义：

  • 访问修饰符 static 数据返回类型 变量名;(推荐)
  • static 访问修饰符 数据返回类型 变量名;

类方法的调用：

  • 类名.类变量名(推荐)
  • 对象名.类变量名

类方法的使用场景：

​ 当你要设计的方法不涉及任何对象相关的成员，则可以将方法设计成静态方法，例如Math类内的大多数方法其实都是static方法。提供一个工具式的功能。

1.6 类方法的详解
  • 类方法和普通方法都随着类的加载而加载，将结构信息存储在方法区内（类变量存放在堆内）
  • 类方法中无this参数，普通函数隐含着this参数
  • 类方法可以通过类名调用，也可以通过对象名调用（与类方法一致）
  • 类方法不允许使用和对象有关的关键字（this && super），但普通方法可以
  • 类方法中只能访问静态变量或静态方法
  • 普通成员方法，既可以访问普通变量，也可以访问静态变量

静态方法，只能访问静态的成员，非静态的方法，可以访问静态成员和非静态成员😁😁😁

三、main方法说明

​ public static void main(String[] args)

1. main方法语法说明

  • Java虚拟机需要调用类的main()方法，所以该方法的访问权限必须是public

  • Java虚拟机指向main()方法时不必创建对象，所以为static

  • 该方法接受Sting数组参数，该数组保存执行Java命令时传递给所运行的类的参数

    ​ args数组存储的是main后的一系列数据。

  • Java执行的程序 参数1 参数2 参数3

  • main方法可以访问本类的静态成员（因为它也是静态方法），所以main方法如果想要访问本类的非静态成员需要创建对象。

2. main方法的动态传值

四、 代码块

1. 代码块概念

​ 是一种没有方法名、返回值、参数，只有方法体的代码，它在类的加载时或者创建类对象时隐式调用，又称为初始化块

2. 基本语法

[修饰符]{ 代码 };

2.1 基本说明
  • 修饰符可选，要写的话，也只能写static
  • 代码块分为两种，一种为static修饰的静态代码块，无修饰的普通代码块
  • 逻辑语句是（输入，输出，方法调用，循环，判断等）
  • ;号可以写上，也可以省略

3. 代码块的益处

  • 相当于另外的一种形式的构造器（对构造器的补充），可以做初始化的操作
  • 如果多个构造器中都有重复的语句，可以抽取到初始化块中，提高代码重用性

4. 代码块细节

  1. static代码块也叫静态代码块，它随着类的加载而执行，并且只执行一次；普通代码块，每创建一个对象就执行一次。
  2. 类什么时候被加载？
    • 创建对象实例时
    • 创建子类对象实例，父类也会被加载
    • 使用类的静态成员时（静态属性，静态方法）
  3. 普通代码块，在创建对象实例的时候被隐式的调用一次。 而如果只是使用类的静态成员时，普通代码块不会被执行。
  4. 创建一个对象时，在一个类 调用的顺序是：
    • 静态代码块和静态属性初始化调用的优先级是一样的，如果有多个静态代码块和多个静态变量初始化，则按他们定义的顺序调用
    • 普通代码块和普通属性初始化调用的优先级是一样的，如果有多个普通代码块和多个普通变量初始化，则按他们定义的顺序调用
    • 调用构造方法
  5. 构造方法的最前面其实隐含了super()和调用普通代码块，静态代码块或者静态属性初始化在类加载时就执行完了
  6. 创建一个代码的子类对象时，他们的代码执行流程是：
    • 父类的静态方法，父类的静态属性初始化
    • 子类的静态方法，子类的静态属性初始化
    • 父类的普通方法，父类的普通属性初始化
    • 父类的构造器
    • 子类的普通方法，子类的普通属性初始化
    • 子类的构造器
  7. 静态代码块只能调用静态成员，普通代码块可以调用任意成员

创建一个对象-》查看该类构造器-》依据构造器调用super-》依据构造器调用父类静态属性和方法-》调用父类

五、单例设计模式

1. 设计模式概念

​ 设计模式是在大量实践中总结和理论化之后优选的代码结构，变成风格，以及解决问题的思考方式，设计模式就像是游戏攻略一样，免得我们自己去跑图找道具。

2. 单例设计模式

  • 类的单例设计模式，就是采取一定的方法保证整个软件系统中，对某个类只存在一个对象实例，并且该类只提供一个取得其对象实例的方法
  • 单例设计模式有：1。饿汉式 2. 懒汉式

3. 饿汉式单例设计模式实现步骤（不管用不用都会给你创建实例）

  • 构造器私有化（防止用户new对象）、
  • 类的内部构建对象
  • 向外暴露一个静态的公共方法
  • 代码实现

4. 懒汉式单例设计模式（你不用，它不创建）

  • 构造器私有化 =》防止new
  • 类的内部创建对象
  • 向外部暴露一个静态共有方法

5. 饿汉和懒汉的区别

  1. 二者最主要的区别是创建对象的时机不同：饿汉式是在类加载创建对象实例，懒汉式是在使用时创建
  2. 饿汉式不存在现场安全问题，懒汉式存在
  3. 饿汉式有浪费资源的可能，懒汉式不会。
  4. javaSE java.langRuntime就是静态的单例模式

五、 Final关键字

1. final的使用前提

  • 当不希望类被继承时
  • 当不希望父类的某个方法被子类覆盖时
  • 当不希望类的某个属性值被修改时
  • 当不希望某个局部变量被修改时

2. final的使用细节

  • 如果一个类已经是final类，那么就没必要再将方法修饰为final方法
  • final不能修饰构造器
  • final和static往往搭配使用让效率更高，底层编译器做了优化处理
  • 包装类几乎都是final，String也是final。

六、抽象类

1. 抽象类介绍

  • 用abstract关键字修饰类的时候，这个类就是抽象类
  • 抽象类的价值更多是在于设计，设计者设计好之后让子类继承并实现抽象类
  • 抽象类只能修饰类和方法，不能修饰其它。

2. 抽象类使用细节

  • 抽象类不能被实例化（特点）
  • 抽象类不一定用到abstract方法，也就是抽象类可以没有抽象方法（特点）
  • 一旦类包含抽象方法，就一定要把它声明为抽象类（语法）
  • abstract只能修饰类和方法不能修饰其他（语法）
  • 抽象类可以有任意成员，并不是说只能有抽象方法（特点）
  • 抽象方法不可以有方法体（语法）
  • 如果一个类继承了抽象类，那么它必须实现抽象类的所有抽象方法，除非它自己也声明为抽象方法（特点）
  • 抽象类不可以用private final static来修饰，因为这些关键字都和重写违背（抽象类的抽象方法需要实现就必须让子类访问或者重写，所以不可以用这些）（语法）

1. 模板设计模式

题目：有多个类内有多个job方法，且需要计算各个类运行的时间

​ 通过建立一个抽象父类的模板，让每一个子类都可以通过父类的culculateTime()方法来计算时间，而该方法内通过调用各个子类的job方法来实现动态绑定计算。这就是模板设计模式

七、接口

1.基本介绍

​ 接口就是把给出的一些没有实现的方法封装在一起，在某个类需要使用的时候根据具体情况把方法写出来。

​ 语法：interface interfaceName{ 属性 方法}

​ class 类名 interfaces 接口{ 自己属性; 自己方法; 必须实现的接口的抽象方法}

定义接口的方法时，方法可以是抽象方法、默认实现方法、静态方法

2. 接口的使用前提

​ 项目经理在管理项目代码质量的时候会编写接口定义代码规范与标准再交给程序员实现接口完成类的编写。

3. 注意事项

  • 接口不能被实例化
  • 接口中的所有方法是 public
  • 接口中抽象方法可以不用abstract修饰
  • 一个普通类实现接口必须把该接口所有方法都实现
  • 抽象类实现接口可以不实现接口的方法
  • 一个类同时可以实现多个接口
  • 接口中的属性，可能是final的。public static final 修饰符。比如：int a = 1;实际上是public static fianl int a =1;
  • 接口中属性的访问形式：接口名.属性名
  • 一个接口不能继承其它的类，但是可以继承多个别的接口
  • 接口的修饰符只能是public 和 默认，这点和类的修饰符是一样的

4. 接口与继承的关系

​ 接口是对继承机制的拓展，接口是比继承更加灵活的存在（继承是is-a，而接口是like-a）

八 . 内部类

1. 内部类概念

​ 一个类的内部又嵌套了一个完整的类的结构，被嵌套的类称为内部类，嵌套再类外部的类称为外部类

2. 基本语法

class Outer{ class Inner{ }} class Other{ }

3. 内部类的分类

  1. 定义再外部类局部位置上（比如方法内）：
    • 局部内部类（有类名）
    • 匿名内部类（无类名）
  2. 定义在外部类的成员位置上：
    • 成员内部类（无static修饰）
    • 静态内部类（有static修饰）

4. 局部内部类

演示：

  • 局部内部类是定义在外部类的局部位置，通常在方法内
  • 可以直接访问外部类的所有成员，包括私有
  • 不能添加访问修饰符，但是可以用final修饰，因为局部变量也可以使用final
  • 作用域仅仅在定义它的方法体内（相当于另一种形式的局部变量）
  • 局部内部类可以直接访问外部类的成员
  • 外部类只能在方法中创建局部内部类的对象，然后调用方法
  • 外部其他类不允许访问局部内部类
  • 如果外部类和局部内部类重名时，默认准寻就近原则，如果下个访问外部类成员，就使用外部类名.this.成员来访问

5. 匿名内部类

1. 说明

​ 本质还是一个类，且是个内部类，该类没有名字（其实有，你看不到,系统命名），匿名内部类同时是一个对象new 类或接口（参数列表）{ 类体 };

2. 演示
3. 细节

  • 匿名内部类的语法比较独特，它既是对象又是类，所以可以调用匿名内部类的方法

  • 可以直接访问外部类的成员，包括私有

  • 不能添加访问修饰符，因为它的地位是局部变量

  • 作用域是所在代码块

  • 匿名内部类—访问—-》外部类成员 直接访问

3. 成员内部类

1. 说明
  • 成员内部类定义在外部类的成员位置并且没有static修饰。
  • 可以添加任意访问修饰符（public protected 默认 private）【它的地位其实就是个成员】
  • 可以访问外部类的所有成员 包括私有
  • 作用域，和外部类的其他成员一样
  • 成员内部类——访问—–>外部类：直接访问
  • 外部类——访问—->内部类：创建对象再访问
  • 外部其他类——访问—->成员内部类：方法1-创建外部类，再用外部类创建成员内部类；方法2-通过多态机制创建一个编译类型为外部类的其他外部类对象。 方法3-直接给外部类弄个方法专门创建成员内部类
  • 外部类和内部类成员重名时，内部类的访问该重名成员遵循就近原则，如果想使用外部类成员，则可以用[外部类名.this.成员]

4. 静态内部类

1. 静态内部类特点
  • 其放在外部类的位置，并且有static修饰（和成员内部类最大的区别）
  • 可以直接访问外部的所有静态成员，包含私有的，但是不能直接访问非静态成员（与静态变量一致）
  • 可以增添任意访问修饰符
  • 作用域：同其他的成员组成一个整个的类体
  • 静态内部类—-访问—-》外部类（比如：静态属性）[直接访问]
  • 外部类—-访问—-》静态内部类 访问方式：创建对象再访问
  • 外部其他类—-访问—-》静态内部类 ：通过类名直接访问（满足访问权限的前提）因为它是个静态的成员，与静态方法相似。
