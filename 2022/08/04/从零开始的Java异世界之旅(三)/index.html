一、枚举

一、枚举是什么

​ 枚举指的是将类的对象按照默认值列举，而不让操作者对类进行不必要的修改；例如：季节类只有春夏秋冬，月份类只有1-12月，粉笔类只有指定的色彩，这些类都不允许你创建一个不符合他们标准的对象，例如创建个什么半春夏，半秋冬之类的，所以枚举就有了。

​ 所以，枚举类是具有特定对象的类

二、枚举类的实现方式

  1. 自定义类实现枚举
  2. 使用enum 关键字实现枚举

三、自定义类实现枚举

  1. 不需要set方法，只用get（因为枚举的对象通常为只读）

  2. 对枚举对象/属性使用final+static共同修饰，实现底层优化

  3. 枚举对象名通常使用全部大写，常量的命名规范

    public final static Season SPRING = new Season("春天","温暖");

  4. 枚举对象根据需要，也可以有多个属性

四、使用enum来实现枚举类

  1. 使用eunm来代替关键字class
  2. public static final Season SPRING = new Season("春天","温暖")改成SPRING("春天"，"温暖")

五、使用细节

  1. 当我们使用enum关键字开发一个枚举类时，默认会继承Enum类（javap反编译获得）
  2. 传统的public static final Season SPRING = NEW Season(“春天”,”温暖”); 转化为 SPRING(“春天”,”温暖”)它调用了哪个构造器？
  3. 如果使用无参构造器创建枚举对象，则实参列表和小括号都可以省略（使用无参构造器可以省略括号）
  4. 当有多个枚举对象时，使用,来间隔，最后一个分号结尾
  5. 枚举对象必须放在枚举类行首

六、enum常用方法

  1. toString ： return name;可重写

  2. name: 返回当前对象名

  3. ordinal: 返回当前对象位置号

  4. values: 返回当前枚举类的（从反编译查看，返回一个数组，值为所有定义的枚举对象）

    for(Season2 season : values)增强for循环

  5. valueOf: 将字符串转化为枚举对象，要求字符串必须为已有的常量名。否则异常

    Season season = Season.valueOf("AUTUMN");在Season中查找是否有AUTUMN的枚举对象，有就返回

  6. compareTo: 比较两个枚举常量，比较的就是位置号

    sout Season.AUTUMN.compareTo(Season.SUMMER);

八、使用细节

  • 使用enum关键字后，就不能继承别的类了，因为enum会隐式继承Enum，而Java是单继承
  • 枚举类和普通类一样可以实现接口

三、注解

一、注解是什么

  1. 注解也被称为元数据，用于修饰包、类、方法、属性、构造器、局部变量等数据信息
  2. 和注释一样，注解不影响程序逻辑，但注解可以被编译运行，相当于嵌入在代码中的补充信息
  3. javaEE中注解可以用来配置应用程序的任意切面，代替javaEE旧版中所遗留的容易代码和XML配置

二、基本的注解介绍

  • @Override:表示重写父类方法，只能用于方法

  • @Depricated: 表示某程序已弃用

    ​ 过时与不能用是两回事，虽然判定过时了，但是还是可以用的

    • ​ 可以修饰方法、类、字段、包、参数等
    • ​ 可以作文版本升级的过渡使用

  • @SuppressWarings: 一致编译器警告

    • ​ 可以不显示警告@SuppressWarings({"all"})这样子使用，里面的all代表不显示所有警告，其中all可以更替为其他警告，但太多了自己查文档
    • ​ 作用范围：与写的位置与放置位置相关，写在main方法内，那么就是整个main方法，直接放在类上面就是整个类
    • 有row

三、JDK的元注解 （了解一下）

定义：修饰注解的注解

  • Retention 指定注解的作用范围，三种 SOURCE,CLASS,RUNTIME
  • Target 指定注解在哪里使用
  • Documented 指定该注解是否会在javadoc体现
  • Inherited 子类会继承父类的注解
