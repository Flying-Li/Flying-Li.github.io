从零开始的Java异世界之旅（一）

一、包

包的本质是创建一个不同的文件夹保存文件

案例1：如果你想创建两个同名文件在一个目录下是不允许的，所以包的存在可以让他们多一个层级

命名规则

  1. 只能包含字母、数字、下划线、小圆点
  2. 开头不能数字
  3. 不能说关键字或者保留字

命名规范

  2. 一般是小写字母.小圆点
  3. com.公司名.项目名.业务模块名

包引用尽量要用啥引用啥

二、访问修饰符

  • public 同类|同包 | 子类 | 不同包
  • protected 同类 | 同包 | 子类
  • private 同类
  • 默认 同类 | 同包
  • 类的修饰符只有 Public 和 默认

三、面向对象

一、封装

面向对象的三大特征： 封装 继承 多态

封装： 把抽象出的数据[属性] 和堆数据的操作[方法] 封装在一起，程序只能通过被授权的操作[方法] 才能对数据进行操作

举例：遥控器对于电视就是封装，你只能通过遥控器来控制里面操作电视的电路来获得操控的反馈（开机关机）但是不可以（无法）直接冲进电视里修改这一切。

封装的好处： 隐藏实现的细节 | 对数据进行验证保证合理性

封装的实现步骤：

  1. 将属性进行私有化

  2. 提供一个公共的set方法，用于属性的判断和赋值

  3. 提供一个公共的get方法，用户获取属性的值

案例： 不能随意查看年龄 工资 等隐私，并且对设置的值进行合理的验证

二、继承

​ 如果我现在有两个类： 小学生类和研究生类，但是他们俩在年龄，姓名等等属性和方法几乎都一样，我们有没有办法去少写点代码给他俩公用？

继承基本语法： class 子类 extends 父类{ }

tips：

  • 子类会自动拥有弗雷定义的属性和方法
  • 父类叫超类（基类）
  • 子类叫派生类

代码案例：com.hspedu.extends包内： Student(父类) Pupil(子类1) Graduate(子类2)

继承细节： ExtendsDetail

  1. 子类继承了父类所有的属性和方法，但是私有属性和方法不能在子类直接访问，要通过公共方法去访问（子类是否继承父类的私有方法？ 是否只能通过公共方法访问）

    ————-子类继承父类所有属性和方法，只不过有的你可以用有的你不可以用而已。

  2. 子类必须调用父类的构造器，完成父类的初始化

    ————-创建子类的对象的时候一定会调用父类的构造器同时子类的也会被调用（子类构造器其实隐藏了一个super来调用父类的构造器，哪怕你创建了子类的对象的有参构造器，它也会去调用父类的无参构造器）

  3. 当创造了子类对象时， 不管使用子类的哪个构造器，默认情况下总会去调用父类的无参构造器，如果父类没有提供无参构造器，则必须在子类的构造器用super去指定使用父类的哪个构造器完成父类的初始化工作，否则编译不通过

  4. 如果希望指定调用父类的某个构造器就用super显示的调用一下

  5. 使用super一定要放在构造器的第一行

  6. super() and this() 都只能放在构造器第一行，因此这两个方法不能共存在一个构造器

    使用方法super(参数列表) | this(参数列表)

  7. 所有的类都是object的子类

  8. 父类的构造器的调用不限于直接父类！将一直往上追溯知道object类

  9. 子类最多只能继承一个父类（指直接继承），即 java中的单机机制

  10. 不可滥用继承，子类和父类必须满足is-a的关系

继承的本质（重要）

案例代码——探讨问题：Son son = new Son();之后，内存发生了什么变化

  1. new完son之后会在方法区开辟对象的空间，并且按照继承的逻辑逐层向上开辟父类空间直至Object类
  2. (1)完成后堆内开辟空间地址为0x11
  3. (2)完成后在0x11内开辟一个空间存放顶层父类的属性值name和hobby，由于该变量的数据类型不是基本数据类型所以其数值为常量池内非基本数据类型数值的地址如0x22 | 0x33
  4. 与(3)相同，将顶层父类的下一层父类的变量也存储在0x11空间内，直至加载完子类
  5. 将0x11赋值给son。

如何从多个同名变量内调用出你想调用的那个变量

  1. System.out.println(son.name)
  2. 系统将按照继承关系，从最底层开始查找有无可访问的name变量
  3. 最开始是son内是否有name，有则输出，无则查father
  4. 注意，如果有该变量但是访问权限不足会报错

  1. new B()调用了B的无参构造器
  2. this(“abc”)调用了B(String name)
  3. B(String name)调用了super()
  4. A()输出了a
  5. B(String name)输出了b name
  6. B()输出了b结束

super关键字

  1. 调用父类构造器，分工明确，父类属性由父类初始化，子类属性由子类初始化
  2. 当子类中有和父类中的成员重名时，访问父类的成员必须用super，没有重名直接使用super、this、直接访问是一样的效果
No. 区别点 this super
1 访问属性 访问本类中的属性，如果本类没有此属性就查找父类是否有该属性 直接访问父类中的属性
2 调用方法 同上 同上
3 调用构造器 调用本类的构造器，必须放在构造器首行 调用父类构造器，必须放在子类构造器的首行
4 特殊 表示当前对象 子类中访问父类对象

方法覆盖

描述：子类中一个方法与父类中的一个方法在方法名、返回类型、参数列表均一致，则称子类覆盖了父类的方法

方法覆盖具有哪些现象？

—-当新建子类的对象的时候，会调用子类的的方法 —-这是为什么？ —-因为按照继承的内存分析，会优先在子类查找该方法

方法覆盖的细节（注意事项）

  1. 子类的方法和参数、方法名称、要和父类方法的参数、方法名称完全一样（方法的定义）

  2. 子类方法的返回类型和父类方法返回类型一样，或者是父类返回类型的子类（结构问题）

    public object n1(){ return null; }（父类） public String n1(){ return null; }（子类） 这样的继承关系下的返回类型是合法的，但是 public String n2(){ return null; }（父类） public object n2(){ return null; }（子类）这样的继承关系下的返回类型不合法。

  3. 子类方法不能缩小父类方法的访问权限

    具体规则同上，访问权限分级： public > protected > 默认 > private

    例如： void sayOK(){ } (父类) public void sayOK(子类) 是合法的;

方法覆盖以及方法重写的区别

名称 发生范围 方法名 参数列表 返回类型 修饰符
重载 本类 同名 类型、个数、顺序至少有一个不同 任意 无要求
重写 父子类 同名 相同 返回类型仅能为父类的下层或一致 子类不能缩小父类

三、多态

一、方法的多态

二、对象的多态

  1. 一个对象的编译类型和运行类型可以不一致
  2. 编译类型在定义对象时确定了，不能改变
  3. 运行类型可以变化的
  4. 编译类型看定义时 = 号左边， 运行类型看 = 号的右边

三、多态的使用注意事项

多态的前提：两个对象或者类之间存在继承关系

多态的向上转型

指的是：父类的引用指向子类的对象

例如： Animal animal = new Dog() 我创建了一个Dog的栈，但是把引用给了Animal，让这个栈升级到了Animal的级别，指向上转型。

语法： Father fatherName = new Son();

特点： 编译类型看左边，运行类型看右边

​ 可以调用父类全部成员（前提是访问权限允许）； 不能调用子类特有成员；最终效果看子类

在编译代码阶段，编译器只能是识别编译代码，而不是按照内存去分析，所以你创建了Animal animal再调用animal.catchMouse()方法编译器认为是错误的，因为catchMouse是Dog的方法，对于编译器来说这是不可以的. 但当你调用了animal和dog都有的eat方法时，编译器认为可以了，因为animal认为有Animal类内有eat方法，尽管按照继承的关系，实际代码运行会先考虑使用dog的eat方法。

多态的向下转型

但是毕竟你引用的对象是Dog对象，里面一定包含着catchMouse方法，如果你一定想要使用这个方法但是编译器不允许怎么办呢？—-这就需要多态的向下转型

语法： Son sonName = (Son) fatherName; 这里的sonName和fatherName指的都是引用名而不是对象名。

注意：

  1. 只能强转父类的引用，不能强壮父类的对象
  2. 要求父类的引用必须指向的是当前目标的对象
  3. 可以调用子类类型的所有成员

多态的细节讨论

  1. 属性没有重写的说法，属性是按编译类型来调用的
  2. instanOf可以用来检测两个对象是否是同一个类或者是其中一个类的子类

Java的动态绑定机制

  1. java的动态绑定是按照运行类型绑定的
  2. 变量是不具有动态绑定效果的，哪里调用就用哪里的

Object类详解

Object 是所有类的父类，所以这个类的方法是所有类都有的

Object类的有哪些常用方法？ —-clone()、equals()、finalize()、getclass()、hashcode()、toString()等

  1. equals方法

== 和 equals的对比

==是一个比较运算符

  1. ==:既可以判断基本类型，也可以判断引用类型
  2. ==:如果判断基本类型，判断的是值是否相等
  3. ==:如果判断引用类型，判断的是地址是否相等，判断的不是 是不是同一个对象

equals

  1. equals:是Object类中的方法，只能判断引用类型
  2. 默认判断的是 地址是否相等，子类往往重写该方法用于判断内容是否相等，比如Integer和String

Hashcode

  1. 提高具有哈希结构的容器的效率
  2. 两个引用，如果指向的是同一个对象，则哈希值肯定一样的
  3. 两个引用，如果指向不同的对象，则哈希值是不一样的
  4. 哈希值主要根据地址来计算的，不能完全将哈希值等价于地址
  5. 后面再集合中，hashcode如果需要的化，也会重写

toString

返回该对象的字符串形式，默认返回：全类名+ @ + 哈希值的16进制，子类往往重写toString用于返回对象信息

return getClass().getName() + @ + Integet.toHexString(hashCode());

重写toString方法，打印对象或拼接对象时，都会自动调用该对象的toString形式

alt + insert 自动生成toString

直接输出一个对象是 toString被默认调用

finalize

  1. 当对象被回收的时候，系统会自动调用该对象的finalize方法，子类可以重写来释放资源
  2. 什么时候垃圾被回收：当某个对象没有任何引用时，则jvm会认为这个对象是垃圾对象，就会使用垃圾回收机制来销毁对象，在销毁之前会调用finalize方法
  3. 垃圾回收机制的调用，是由系统来决定的，也可以通过System.gc()主动触发回收机制
