集合用起来并不是很难，但是我们要是想看源码和理解它的底层很难，最主要的我们要直到我们在什么时候用什么集合

  1. 集合
    • 可以动态保存任意多个对象，使用比较方便
    • 提供了一系列方便的操作对象的方法，add remove set get等
    • 使用集合添加、删除新的元素的代码更简介

集合的框架体系图

首先：Java的集合类分为两大类，一类是单列集合，一类是双列集合

先看单列集合Collection

再看双列集合Map

单列集合和双列集合的不同是什么

  • 单列集合的存放方式是直接存放一个对象
  • 双列集合是通过K-V方式存放的，就是一个键对应一个值，这一点再Hadoop学习里我们的单词解析器有说，那三层的mapper就是通过三重转化K-V来查找数据并输出的

Collection接口 特点和方法

特点

  • Collection实现子类可以存放多个元素，每个元素可以是Object
  • 有些Collection的实现类可以存放重复的元素，有些不可以
  • 有些Collection的实现类，有的是有序的List，有些是无序的Set
  • Collection接口没有直接的实现子类，是通过它的子接口Set和List来实现的

方法

  • add

  • remove

    删除的元素是按序号排的，序号从0开始list.remove(0)

    也可以指定删除某个元素list.remove("jack")之类的里面填元素的值

  • contains

    list.contains("jack")返回值是一个boolean值

  • size

  • isEmpty

  • clear

  • addAll

    这个addAll是把另外一个集合的元素整体添加到调用者list内

    list.addALL(list2);把list2全加list里去

  • containsAll

    list.containAll(list2)同上

  • removeAll

    同上

遍历集合

  1. 方法一、使用Iterator对象

    只要实现了Collectio接口的集合类里都有Iterator方法以返回一个iterater接口的对象

    iterator仅用于遍历对象，不存放对象

    当我们用iterator获取一个对象的时候我们就获取了一个存放数据的栈，iterator.next()会扫描栈内元素，如果有下一个就返回该元素，直到没有下一个为止

    调用iterator.next()之前一定要检查是否有hasNext，不然会报错

    当我们使用iterator遍历了一个集合之后，如果我们再之后再使用iterator.next就会报错，因为这个迭代器的指针没有复原，依然指向栈底，我们要使用iterator = col.iterator()重置这个弯钩游标才会给你指回去

  2. 方法二、增强for循环

    这个方法用的更多，还可以用到数组上，很不错

    增强for的底层还是迭代器iterator，源码上还是调用了各种next啥的，可以当成简化版本的迭代器

List接口 特点和常用方法

特点

  • list集合类中的元素是有序的（添加顺序和取出顺序一致）而且，可以重复

  • List集合类的每个元素都有对应的顺序所以，即支持索引

    通过list.get(3)方法可以通过索引取值

  • List容器中的元素都对应一个整数型的序号记载其再容器的位置，根据序号我们可以去除容器中的元素

方法

  • add(int index,Object element)

  • addAll(int index,collection elements)

  • get(int index)

  • indexOf(Object obj):返回obj在集合里第一次出现的位置，返回int

  • lastIndexOf(Object obj)：返回obj在集合里最后一次出现的位置，返回int

  • remove(int index)

  • set(int index,Object element)：替换某个位置的元素

  • subList(int fromIndex,int toIndex)

    如果删除一个元素就相当于整个List序号可能会变哦

遍历

前两种就是collectionn的遍历，还有一个就是普通的获取list.size 然后list.get的普通for循环遍历

ArrayList底层源码和结构

  1. 底层结构

    • ArrayList可以存放任何参数，包括null
    • ArrayList是由数据来实现的数据存储
    • ArrayList基本等同于Vector，除了ArrayList是线程不安全的（执行效率较高），在多线程的情况下不建议使用ArrayList

  2. 源码分析

    • ArrayList种维护了一个Object类型的数组elementData
    • 在创建ArrayList对象时，如果使用的无参构造器，则初始elementData的容量为0，第一次添加，则扩容elementData为10，如果需要再次扩容，则扩容elementData为1.5倍（扩容机制——无参构造器）
    • 如果使用指定大小的构造器构造ArrayList，则elementData容量为指定大小，如果需要扩容，则直接扩容elementData为1.5倍（扩容机制——指定大小构造器）

    扩容机制的底层是如何实现的？

    当我们new ArrayList()时会进入无参构造，它创建了一个elementData = DEFAULT…….（默认的一个数字，大小为0，是空数组)

    此时我们给list添加数据list.add(i)在add方法里会确认你的空间够不够，它会比较你的最小容量和默认容量大小怎么变，JDK8里面是10（默认）

    算了还是详细点吧😒😒，以JDK11为例

    • 进入ArrayList()无参构造器：创建了一个空的elementData数组
    • 进入add方法内执行list.add：先看是否要扩容，进入一个ensureCapcityInternal方法
    • 进入ensureCapcityInternal判断是否要扩容：判断elementData是否是空数组，如果是，就给一个最小容量10，再次确认是否要扩容，用ensureExplicitCapacity方法判断
    • 进入ensureExplicitCapacity判断是否要扩容：用最小容量减去当前数组(elementData)实际容量是否大于0，是的话就进入gorw方法进行扩容，够了就不扩容
    • 进入grow方法进行扩容：使用扩容机制来确定要扩容到多大，如果是第一次就newCapacity=10；如果十个满了的话就按照当前数组大小的1.5倍扩容，通过Array.copyOf来给elementData赋予newCapacity的容量并且保留原先的数据（copyOf的特点就是保留数据，所以使用这个）
    • 一步一步返回到最初的add方法，这时候elementData已经编程10个空间，然后就可以添加元素了
    • 然后就继续循环了

    啊！源码好弯钩麻烦！🤦‍♀️🤦‍♀️

Vector源码和结构

  1. 底层结构

    • Vector底层也是一个对象数组，是个elementData
    • Vector是线程同步的，Vector类的操作方法带synchronized
    • 如果需要考虑线程安全，优先使用Vector
    • 安全，但是效率不高
    • 扩容是按2倍扩容，默认大小为10

  2. 源码分析

    源码和ArrayList几乎是一致的，就不过多赘述了

LinkedList源码和结构

  1. 底层机制

    • LinkedList底层实现了一个双向链表和双端队列特点
    • 可以添加任意元素，元素可重复且元素可为null
    • 线程不安全，没有实现同步
    • LinkedList种维护了两个属性first和last分别指向首节点和尾节点
    • 每个节点里面又维护了prev、next、item三个属性，其中prev指向前一个节点，next指向下一个节点，实现双向的链表
    • 所以LinkedList的元素的添加和删除不是通过数组完成的，相对来说效率较高
    • 模拟一个简单的双向链表

  2. LinkedList的增删改查案例（增）

    这部分就是数据结构的内容了，只是把伪代码转换成了Java代码的内容并且写成LinkedList的内容，交代一下大概原理并且如何使用吧

    • 我们new LinkedList()会生成一个大小为0的数组
    • 然后我们调用list.add(1)，在底层我们会给first一节点和尾节点默认为null，现在我们加入一个值，我们把1添加进链表空间的中间，pre值和next值都是null，first和last都是它自己
    • 现在我们再加入一个节点，我们的next就会变为二节点的地址，二节点的内容也是一节点的地址
    • 这部分基本就是数据结构的内容了（感谢数据结构教会我这些让我现在理解的没那么痛苦）

  4. 和ArrayList的比较

    Linked的增删效率很高，通过链表最佳，但改查的效率较低

    • 如果我们改查操作比较多，那么就ArrayList
    • 如果增删的内容比较多，那么就用LinkedList
    • 一般来说在程序里百分之九十都是查询，大部分情况下我们都是ArrayList
    • 根据业务的灵活性，我们选择性使用，看你自己

Set接口 特点和常用方法

特点

  • 无序（添加和取出的顺序不一致），没有索引
  • 不允许重复元素，所以最多包含一个null
  • JDK API种Set接口的实现类有：AbstractSet,ConcurrentSkipSet、HashSet、TreeSet

常用方法

与List接口一样，Set接口也是Collection的子接口，因此方法跟Collection接口一样

遍历方法

  • 可以使用迭代器
  • 增强for
  • 不能使用索引，因为没有索引

代码

  • 如果你添加了两个一致的数据，你取出来只能取到一个
  • 添加的数据顺序和取出的数据顺序是不一致的，但是取出的顺序是固定的，虽然是乱的，但是你取出一万次顺序都是一样的

HashSet

  1. 底层结构

    • HashSet实现了Set接口
    • HashSet实际上是HashMap
    • 可以存放null值，但只能有一个
    • HashSet不保证元素是有序的，取决于hash之后再确定索引的结果
    • 不能有重复的元素
    • 在执行add的时候，如果add成功会返回一个boolean值

  2. 源码分析

    上面可以理解，那现在呢？

    HashSet的源码非常复杂，所以篇幅比较长

    首先我们要HashSet的底层是HashMap，HashMap的底层是(数组+链表+红黑树)

    当它的链表到达一定的量且数组的大小达到一定的规模，链表就会被树化，变成一个红黑树

数组链表模拟

HashSet的扩容机制

  • HashSet的底层是HashMap
  • 添加一个元素时，先得到hash值，hash值转化为一个索引值
  • 找到存储数据的table，看到这个索引位置是否存放有元素
  • 如果没有，直接加入
  • 如果有，调用equeal比较，如果相同，放弃添加，如果不同，添加到最后
  • 在Java8中，如果一个链表的元素超过八，数组的元素超过64，就会进行树化（卧槽太JB帅了这个词）

LinkedHashSet

  1. 底层机制
    • LinkedHashSet 是HashSet的子类
    • LinkedHashSet 底层是一个LinkedHashMap，底层维护了一个数组+双向链表
    • LinkedHashSet 根据元素的 hashCode值来决定元素存储的位置，同时使用链表维护元素的次序，使元素看起来是以插入顺序保存的
    • LinkedHashSet 不允许添加重复元素
    • LinkedHashSet 的每一个节点有pre和next属性
    • LinkedHashSet 在添加一个元素时先计算hash求得索引，再确定该元素再hashtable的位置，然后将添加的元素加入到双向链表，如果已经存在就不加了
    • LinkedHashSet 遍历的化确保了插入顺序和取出顺序一致
  2. 源码分析
    • 第一次添加时，table第一次添加会扩容到了16
    • 初始化的table数组是HashMap$Node[ ]存放的数据是LinkedHashMap$Entry类型
    • 这里还是走了putVal()方法，麻烦

Map接口

Map接口实现类的特点

  • Map和Collection并列存在，哟关于保存具有映射关系的Key-Value

  • Map中的key和value可以是任何引用类型的数据，会封装到HashMap$Node对象去

  • Map中key不许重复，原因和HashSet一样，因为key的本质是用hashCode做的，如果重复会有重复的hash，所以不中

    如果你添加了一个重复key，那么后来者的value会替换前者

  • Map中的value可以重复

  • Map的key可以是null，value也可以是null，但key为null只能有一个，value为null可以多个

  • 常用String类作为Map的key

  • key和value之间存在单项的一对一关系，即指定的key总能找到value

  • Map存放数据的k-v是放在一个Node中的，因为Node实现了Entry接口，书上也说过，一对k-v就是一个Entry

    所谓的Set接口也好Collection接口也好，他们的只是再table里面建立了一个通往Node节点的引用，而Node里是用Key-Value，我们的Set或者是Collection是把table里面的一组对象存在了一个Entry里面，而Entry又放在一个EntrySetsou里面，
