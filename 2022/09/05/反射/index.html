反射

简介

🤨🤨提问：为什么我们需要反射，反射有什么用呢？

现在我们有一个配置文件，文件内记载了一个类的路径，以及类中的一个方法，那么我们该如何根据这个配置文件来调用配置文件说明的这个方法呢？

—- 我现在想到了一个方法，用properties的IO流读取配置文件来实现这个步骤

🎏🎏提示：在框架的学习中，几乎需要很多很多的配置文件都有这样的需求，那么我们该怎么做呢？

如果这样的需求能实现该多方便，我们不需要修改一个代码，只需要一个文件就可以控制程序，同样符合我们的开闭模式的设计模式，多好，但是该怎么实现呢？

那就是反射机制！

🙀🙀震惊：居然真的可以输出Cat内的方法hi

🎏🎏提示：在传统的方法里，我们用cat.hi来调用方法，但是当我们想用cat.cry()而不用cat.hi()的时候我们要去源码里面修改hi为cry，但是有了反射的机制，我们发现我们只需要把文件的hi改成cry，程序就自动执行cry了，不用再去修改源码了，我们对修改关闭了，对拓展开放了

反射机制

  1. 反射机制允许程序在执行之间借助于Reflection API取得任何类的内部信息（比如成员比哪里，构造器，成员方法等等），并能操作对象的属性及方法。反射在设计模式和框架底层都会用到
  2. 加载完类之后，在堆中就产生了一个Class类型的对象（一个类只有一个Class对象，这个对象包含了类的完整结构信息，通过这个对象得到类的结构，这个对象就像一面镜子，透过这个镜子看到类的结构，所以我们把它叫做反射

Java反射机制示意图

😵‍💫😵‍💫解释：Java在程序中有三个阶段：编译阶段、加载阶段、运行阶段

编译阶段：我们编辑的.java文件通过javac编译为.class文件

类加载器：.class文件通过类加载器，在堆中使用了一块区域生成了Class类的对象。

加载阶段：这个对象中以数组的形式存储着成员变量、构造器、成员方法,这也意味着这些其实都变成了对象一样的存在形式

运行阶段：这时候我们新建一个cat并且调用它的hi方法，就是我们正常创建、操作一个对象的流程，但同时，我们的cat对象也直到，它是属于哪个Class对象的，而根据cat对象对它所属对象的存在形式，我们可以应用反射。

😵‍💫😵‍💫错乱：就好像编译阶段是个本体，加载阶段是一面镜子，运行阶段是镜子里的虚像，我们通过虚像同样可以看清一个类的本来面目。

反射机制可以完成

  1. 在运行时判断任意一个对象所属的类
  2. 在运行时构造任意一个类的对象
  3. 在运行时得到任意一个类所具有的成员变量和方法
  4. 在运行时调用任意一个对象的成员变量和方法
  5. 生成动态代理

反射相关的主要类

  1. java.lang.Class:代表一个类，Class对象表示某个类加载后在堆中的对象

  2. java.lang.reflect.Method:代表类的方法

  3. java.lang.reflect.Field:代表类的成员变量

  4. java.lang.reflect.Constructor:代表类的构造方法

反射的优点和缺点

  1. 优点：动态的创建和使用对象，使用灵活，没有反射就没有框架

  2. 缺点：使用反射基本是解释执行，堆执行速度又影响

    那么我们可不可以优化一下这个速度呢？

反射调用优化——关闭访问检查

  1. Method和Field、Constructor对象都有一个setAccessible()方法

  2. setAccessible作用是启动和禁用访问安全检查的开关

  3. 参数为true表示反射的对象在使用时取消访问检查，提高反射效率。参数值为false则表示启动访问检查

    使用方法hi.setAccessible(true);从1530变成1357很不错

Class类

  1. Class类也是类，因此也继承Object类、

  2. Class类对象不是new出来的，而是系统创建的

    通过ClassLoader的loadClass方法来生成的；

  3. 对于某个类的Class类对象，在内存中只有一份，因此类只加载一次

  4. 每个类的实例都会记得自己是由哪个Class实例所生成的

  5. 通过Class可以完整的得到一个类的完整结构

  6. Class对象是存放在堆的

  7. 类的字节码二进制数据，是放在方法区的，有的地方成为类的元数据（Cat类的字节码二进制数据）

Class常用方法

哪些类型是有Class对象的

  • 外部类，成员内部类，静态内部类，局部内部类，匿名内部类
  • 接口
  • 数组
  • 枚举
  • 注解
  • 基本数据类型
  • void

类加载

基本说明

反射实现了类的动态加载

  1. 静态加载：编译时加载相关的类，如果没有则报错，依赖性太强

    🎏🎏解释：在上述代码中，不论我们输入的是不是1，Cat这个类都会被加载，而如果程序中根本没有Cat这个类则直接报错，因为我们正常使用一个类是静态加载的。

  2. 动态加载：运行时加载需要的类，如果运行时不用该类，则不报错，降低了依赖性

    🎏🎏解释：这时候即使我们输入1，cat这个类也不会被加载，除非我们触发反射机制才给你加载，这就是动态加载

类加载时机

  1. 当创建对象时（new）//静态加载
  2. 当子类被加载时，父类也加载 //静态加载
  3. 调用类中的静态成员时 //静态加载
  4. 通过反射 //动态加载

类加载过程

类加载的三个阶段

💻💻原理：加载阶段：将类的class文件读入内存，并为之创建一个java.lang.Class对象。此过程由类加载器完成

Linking阶段：验证：对文件的安全性进行校验；准备阶段：对静态变量初始化并分配内存空间；解析阶段：把符号引用改为直接引用；总的来说这个阶段将类的二进制数据合并到JRE里面中

初始化阶段：JVM负责对类进行初始化，这里主要指静态成员（执行Java代码）

前两个阶段是JVM机负责的，而第三阶段就是我们可以操控的内容了

Loading阶段

JVM在该阶段的主要目的是将字节码从不同的数据源转化为二进制字节流加载到内存中，并生成一个代表该类的java.lang.Class对象

即：生成一个方法区（里面是类的字节码），以及在堆里面加载类的Class对象

Linking阶段

  1. 目的是为了确保Class文件的字节流中包含的信息符合当前虚拟机的要求，并且不会危害虚拟机自身的安全
  2. 包括：文件格式验证、元数据验证、字节码验证、符号引用验证
  3. 可以考虑使用-Xverify:none 参数来关闭大部分的类验证措施，缩短虚拟机加载类的时间

连接阶段：准备

  1. JVM在该阶段对静态变量，分配内存并默认初始化（对应数据类型的默认初始值，如0、0L、null、false），这些变量所使用的内存都将在方法去中进行分配

    🎏🎏解释：如果成员变量不是静态变量，那么在这个过程是不会给非静态变量分配内存的

    ​ 如果成员变量是静态变量，那么就给该变量分配内存，赋值为默认值（null或者0之类的）

    ​ 如果成员变量是静态常量，那么直接分配内存空间并且赋值

连接阶段：解析

  1. 虚拟机将常量池内的符号引用替换为直接引用过程，叫解析

    🎏🎏解释：在我们的加载阶段如果有两个类，A引用B，这时候会用符号地址（虚拟地址）来描述二者关系

    ​ 而现在，我们加载这个类，我们就把虚拟地址转换为了物理地址，并且把二者的关系用物理地址的方式指向

Initialization（初始化阶段）

  1. 到初始化阶段才真正意义上的开始执行类中定义的Java程序代码，此部分是执行()方法的过程

  2. ()方法是由编译器按语句在源文件中出现的顺序，依次自动收集类中所有静态变量的赋值动作的静态代码块中的语句，并进行合并

    🎏🎏解释：我们定义了一个类，包含静态代码块和静态变量，那么在这个阶段会生成一个client(){ }类，这个类里按顺序收集静 态方法、属性的值，并且合并，然后执行这个合并后的东西；（这玩意会自动加载，有同步机制）

  3. 虚拟机会保证一个类的()方法在多线程环境中被正确的加锁、同步，如果多个线程同时去初始化一个类，那么只有一个线程去执行这个类的()方法，其他线程都需要阻塞等待，直到活动线程执行()方法完毕

通过反射获取类的信息（一大堆的API）

反射暴破创建实例

如何通过反射创建对象

  1. 方式一：调用类中的public修饰的构造器
  2. 方式二：调用类中的指定构造器

Class类相关方法

  • newInstance:调用类中的无参构造器，获取对应类的对象
  • getConstructor(Class…clazz):根据参数列表，获取对应的构造器对象
  • getDecalaredConstructor(Class…clazz):根据参数列表，获取对应的构造器对象

Constructor相关方法

  • setAccessible:暴破
  • newInstance(Object…obj):调用构造器

通过反射访问类中的成员

  1. 根据属性名获取Field对象
  2. 暴力破解：f.setAccessible(true);
  3. 访问f.set(o，值) sout(f.get(o));
  4. 如果是静态属性，则set和get参数o，可以为null

反射访问类中的方法

  1. 根据方法名和参数列表获取Method方法对象：Method m = clazz.getDeclaredMethod(方法名,xxx.class);
  2. 获取对象Object o = clazz.newInstance()
  3. 暴力破解：m.setAccessible(true)
  4. 访问：Object returnValue = m.invoke(o,实参列表)
  5. 如果是静态方法，则invoke的参数o,可以写为null
