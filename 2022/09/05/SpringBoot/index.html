SpringBoot：快速入门？

什么是Spring？

一个简化开发的解决企业级应用开发的复杂性而创建的

Spring如何简化Java开发？

  1. 基于POJO的轻量级和最小侵入性变成；
  2. 通过IOC，依赖注入（DI）和面向接口实现松耦合
  3. 通过AOP和惯例，进行声名式编程
  4. 通过切面和模板减少

SpringBoot是什么？

​ SpringBoot本身并不提供Spring框架的核心特性以及扩展工具，只是用于快速、敏捷的开发新一代基于Spring框架的应用程序，也就是说它不是来替代Spring的，而是和Spring框架紧密结合用于提升Spring开发者体验的工具；SpringBoot也是约定大于配置，默认帮我们进行了很多的设置，多数SpringBoot应用只需要很少的Spring配置，同时它还集成了大量的第三方库配置

​ 简单来说：SpringBoot其实不是什么新框架，它默认配置了很多框架的使用方式，就像maven整合了jar包，springBoot整合了所有框架

🎏🎏解释：这就像是我们maven导入jar包只需要弄个依赖就可以直接完成一样，SpringBoot也许能实现一行代码直接解决SSM里面那么多奇奇怪怪的配置

微服务是什么？

微服务是一种架构，就像MVC三层架构，MVVM架构一样，微服务也是一种架构

🎏🎏解释：微服务把service、dao、controller之类的这种东西变成一个模块，每个模块分在不同的电脑通过http交互，形成一个网状一样的结构

单体应用架构

all in one所有的东西都配置在一起，然后打成一个war包运行在服务器上，但是这种方式如果你需要改动一个很小的改动都需要我们停掉服务器去重新打一个war包上线到服务器上

微服务架构

one for all，把一个应用的很多地方都划分成模块分布在不同的地方，这样我们如果需要修改一个小模块，我们就可以把缓存隔离出来先用着，等我们修改好这个小地方再上线连接到各个地方

理解pom.xml

是怎么做到这么牛逼这么便捷的，我看看pom.xml里面到底导入了什么神圣

  • 一些基本的元信息：groupID、artifactld、version、name、description等
  • parent：继承parent的依赖管理，控制版本和打包内容（这里的路径是网上的路径）
  • dependencies：项目的具体依赖
    • spring-boot-starter-web实现http接口（该依赖中包含了SpringMVC），官网对它的描述是：使用SpringMVC构建web，使用tomcat作为嵌入式容器
    • spring-boot-starter-test用于编写单元测试的依赖包
    • build默认使用spring-boot-maven-plugin，配合parent来把SB应用打成jar包

😵‍💫😵‍💫自言自语：感觉几乎大多数部分都是parent里面做的，我也没导入啥嘞

SpringBoot原理

自动配置：

pom.xml

  • spring-boot-dependencies:核心依赖在父工程中！
  • 我们写入或者引入一些Springboot依赖的时候不需要指定版本，因为有这些版本仓库

启动器：

  • ```xml

    org.springframework.boot spring-boot-starter

    就会帮我自动导入we环境所有的依赖

  • springboot会将所有的功能场景，都变成一个个的启动器

  • 我们需要使用什么功能，就只需要找到对应的启动器：一个个的starter

主程序：

  • SpringBootApplication 标注这个类是一个springboot应用
  • SpringBootConfiguration : springboot的配置
    • configuration: spring配置类
    • Component: 说明这是个spring组件
  • EnableAutoConfiguration: 自动配置
    • AutoConfigurationPackage: 自动配置包
      • Import(AutoConfigurationPackges.Register.class): 自动配置，包注册
    • Import(AutoConfigurationImortSelector.class): 自动配置，导入选择

结论：springboot所有自动配置都是在启动的时候扫描并加载：spring.factories所有自动配置类都在这里面，但不一定生效，要判断条件是否成立，只要导入了对应的start，就有了响应的启动器，我们自动装配就会生效，然后配置成功

  1. springboot在启动时，从类路径下/META-INF/spring.factores获取制动的值
  2. 将这些自动配置的类导入容器，自动配置就会生效，帮我们进行自动配置
  3. 以前我们需要手动配置的东西，现在springboot帮我们做了
  4. 整合javaEE，解决方案和自动配置的东西都在spring-boot-autoconfigure-2.2.0 RELEASE.jar这个包下
  5. 它会把所有需要导入的组件，以类名的方式返回，这些组件就会被添加到容器；
  6. 容器中也存在许多的xxxAutoConfiguration文件，就是这些类给容器中导入了这个场景需要的组件

💻💻解释：自动装配指的是我们往pom.xml内装配的内容被加载，以前我们手动配置的spring-mybatis、srpring-servlet都只需要通过自动装备导入容器就可以帮我自动配置，非常方便，但是这里面的原理很复杂，后续需要恶补
