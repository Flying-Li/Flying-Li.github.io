JavaWeb

1.1、 基本概念

  • web，网页的意思

  • 静态web

    • html,css
    • 提供给所有人看的，数据始终不会发生变化

  • 动态web

    • 提供给每个人的数据始终都会变化的
    • 现在几乎所有网站几乎都是动态web
    • 技术栈：Servlet/JSP，ASP，PHP

    在Java中，动态web资源开发技术统称为web

1.2、web应用程序

web应用程序：可以提供浏览器访问的程序

  • a.html，b.html…多个web资源，这些资源都可以被外界访问，提供服务
  • 我们能访问的任何一个页面或资源，都存在于这个世界上的某一个角落的计算机上
  • 这些个所有的web资源被放在同一个文件夹下组成了我们的一个web应用程序->部署在tomcat服务器上
  • 一个web应用由多部分组成（静态web，动态web）
    • html css js
    • jsp servlet
    • java
    • jar
    • 配置文件

web应用程序编写完毕后，若想提供给外界访问，需要一个服务器来统一管理

1.3、 静态web

  • .htm,.html都是网页的后缀，服务器上如果存在这些东西，那么就一定可以被访问读取
  • 静态web的缺点
    • 轮播图、点击特效：伪动态，通过↓这几种技术
    • JavaScript
    • VBScript
  • 静态web的执行流程：
  • 它无法和数据库交互，数据无法持久化，用户无法交互

1.4、 动态web

页面展示效果每个人都不一样，因人而异

缺点

  • 加入服务器的动态web资源出现错误，我们需要重新编写一个后台程序重新发布（网站维护）

优点：

  • 能与数据库交互（数据持久化，我们可以注册，商品信息更新）
  • 每个用户看到的东西都不一样

4、Http

4.1、什么是HTTP

http是超文本传输协议

  • 文本：html，字符串，
  • 超文本：图片，音乐，视频，定位
  • 80（端口）

https：安全的

  • 443（端口）

4.2、两个时代

  • http1.0
    • HTTP/1.0：客户端与服务器的连接一次只能得到一个资源，然后断开连接，如果你像再次获得这个资源，你只能再次请求
  • http2.0
    • HTTP/1.1

4.3、Http请求

  • 客户端像服务器发起一次请求

    1、请求行

    • 请求行中的请求方式:GET
    • 请求方式：Get Post HEAD.DELETE
      • get: 请求能够携带的参数比较少，大小有限制，会在地址栏的URL内显示，不安全，但高效
      • post：请求能够携带的参数无限制，大小没限制，不会在地址栏显示，安全，较慢（无伤大雅，因为限制都5G了）

    2、消息头

4.4、Http响应

  • 服务器像客户端发起一次响应

    1、 响应体

    2、 响应码

    • 200：请求响应成功

    • 4**： 找不到资源

      • 403：禁止访问

      • 资源不存在

    • 3**： 请求重定向

      • 重定向

    • 5**： 服务器代码错误 500

      • 502：网关错误

    常见面试题

    当你浏览器输入地址栏地址并回车的返回页面的一瞬间，发生了什么？

7、Cookie、Session

什么是cookie？什么是session？

7.1、会话

会话：用户打开一个浏览器，点击很多超链接，打开很多资源，关闭浏览器，这称为一次会话

有状态会话：一个同学来过教室，下次再来，我们知道他曾经来过

  • 一个网站，如何证明你来过？例如我登入一个网站，我隔天进去，不用再登入了，它怎么知道我来过？

    1. 服务端给客户端一个cookie，客户端下次访问带上cookie就可以了

    2. 服务器等级你来过了，下次你来的时候我匹配

7.2、保存会话的两种技术

cookie

  • 客户端技术（响应、请求）

session

  • 服务端技术，可以保存用户使用的session，保存再会话中

实现一个简单案例：服务端告诉你来的时间，下次你登录把这个时间告诉你

7.3、cookie

  1. 从请求中拿到cookie信息

  2. 服务器响应给客户端cookie

  3. 那么一个网站可以存少个cookie

    • 每个cookie只能保存一个信息
    • 一个站点大约存在20个cookie
    • 300个cookie是浏览器的上线
    • cookie大小上限为4kb

  4. 删除cookie

    • 给cookie设置有效期为0，关闭浏览器就没了
      • 设置一个同名cookie（cookie的键唯一）
      • 设置声明周期为0
      • 返回给客户端
      • 这样客户端一被返回这个cookie这个cookie立刻过期

  5. cookie如何传递中文参数？

    • 根据LastLoginTime一样传递
    • 根据URLEncode(“旅社”,”utf-8)来加密，根据URLDecode(“utf-8”)来解码
