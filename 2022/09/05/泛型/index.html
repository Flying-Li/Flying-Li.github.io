泛型

前言

为什么我们需要泛型？

因为如果我们使用arrayList集合的时候如果不加一个泛型来约束，即使我们添加的类型是不一致的，编译器也无法发现这个问题；

好处

  • 不能对加入到集合ArrayList中的数据类型进行约束
  • 遍历的时候，需要进行类型转换，如果集合中的数据量过大会影响效率

泛型介绍

😵‍💫😵‍💫解释：泛型就是一种表示数据类型的数据类型，在以往的编码里，int a = 1那么1就是a，如果String a = “Lemming”那Lemming就是String，但是有了泛型E = Integer那E就是Integer，E = “Lemming”,那E就是String，它面对传来的数据不同有着不同的数据类型，这就是泛型，当然用的时候你还是要指示出来你要什么样的类型的。

  • 泛型又称参数化类型，解决数据类型安全的问题

  • 在类声明或实例化时只要指定好需要的具体的类型即可

  • Java泛型可以保证如果程序在编译时没有发出警告，运行时就不会产生类型转换错误

  • 泛型的作用：可以在类声明时通过一个标识表示类中某个属性的类型，或者某个方法的返回值的类型，或者参数的类型

    🤿🤿自言自语：弹幕说这就好像一个模板一样往里套就好了，还有弹幕说，泛型的内容在语言层面其实是没有编译的，只是编译器通过泛型可以按照泛型的规定来

例题

自定义泛型

基本语法

class 类名 <T,R…>{ }

注意细节

  • 普通成员可以使用泛型（属性和方法）
  • 使用泛型的数组，不能初始化
  • 静态方法中不能使用类的泛型
  • 泛型类的类型，是在创建对象时确定的（因为创建对象，需要指定确定类型）
  • 如果在创建对象时，没有指定类型，默认为Object

自定义泛型接口

基本语法

interface 接口名 <T,R…>{

}

注意细节

  • 接口中，静态成员也不能使用泛型
  • 泛型接口的类型，在继承接口或者实现接口时确定
  • 没有指定类型就默认为Object

自定义泛型方法

基本语法

修饰符<T,R..> 返回类型 方法名（参数列表）{ }

注意细节

  • 泛型方法，可以定义在普通类中，也可以定义在泛型类中
  • 当泛型方法被调用时，类型会确定
  • public void eat(E e){ },修饰符后没有<T,R..>eat方法不是泛型方法，而是使用了泛型
